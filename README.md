# Python 卡通濾鏡影像處理

## 摘要

本專題以 Python 實作卡通化影像處理技術，透過灰階轉換、邊緣偵測、顏色量化與渲染合成，將真實影像轉化為卡通風格圖片。程式採用物件導向設計，以獨立類別封裝各個處理步驟，提供清晰易維護的架構。

## 研究目標

1. 以 OpenCV 庫實作影像處理，學習基礎卡通化演算法。
2. 分別進行灰階轉換、邊緣偵測、顏色量化等多個處理階段，展示各階段的中間結果。
3. 透過中值濾波與自適應閾值處理，提升邊緣偵測品質。
4. 使用雙邊濾波與 K-Means 顏色量化，達成顏色區塊化效果。
5. 最終合成邊緣與顏色，產生專業卡通化結果。
6. 提供適合課堂實驗與作業展示的影像處理示範程式。

## 系統設計

本專案以單一 Python 程式組織，核心流程透過四個主要類別實現：

### 1. 影像載入 (`ImageLoader`)

- 讀取指定路徑的影像檔案
- 驗證檔案存在性，不存在則拋出異常提示
- 支援 OpenCV 支援的所有格式（PNG、JPG、BMP 等）

### 2. 灰階轉換 (`GrayConverter.to_gray()`)

- 使用亮度加權公式將 RGB 影像轉換為灰階
- 公式：`gray = 0.299R + 0.587G + 0.114B`（OpenCV 內建）
- 為後續邊緣檢測提供單通道基礎

### 3. 邊緣偵測 (`EdgeDetector.detect()`)

- **中值濾波**：使用 5×5 核心進行中值濾波，去除雜訊同時保留銳利邊緣
- **自適應閾值**：應用自適應閾值處理（block size=9, C=5），適應影像局部亮度變化
- 輸出二值邊緣圖，白色（255）表示邊緣，黑色（0）表示背景

### 4. 顏色量化 (`ColorQuantizer.quantize()`)

- **雙邊濾波**：多次（3次）應用雙邊濾波器 (kernel=9, σ_color=75, σ_space=75)
  - 平滑色塊，保留銳利邊界
- **K-Means 聚類**：將原始影像顏色聚類至指定數量（預設16色）
  - 條件：最多20次迭代，終止誤差 = 0.001
  - 輸出色塊化的影像

### 5. 卡通化渲染 (`CartoonRenderer.render()`)

- 使用 bitwise_and 操作將量化後的顏色與邊緣遮罩結合
- 邊緣處設為黑色墨線，色塊區域保留顏色量化結果
- 最終產生卡通風格影像

## 操作流程

1. 準備輸入影像 `image.png`（或指定其他路徑）
2. 執行程式主函數
3. 程式自動進行四個處理步驟，並輸出中間結果
4. 最終卡通化影像儲存為 `step4_final_cartoon.png`
5. 檢視各階段輸出檔案，觀察處理效果

## 時間複雜度分析

令：
- `w`、`h` = 影像寬、高
- `n` = 像素總數 = `w × h`
- `k` = K-Means 聚類數（預設16）

### 灰階轉換

- 遍歷每個像素進行亮度加權轉換
- 複雜度：`O(w × h)`

### 邊緣偵測

- 中值濾波：`O(w × h × 25)`（5×5 核心）
- 自適應閾值：`O(w × h × 81)`（9×9 區塊）
- 合計：`O(w × h)`

### 顏色量化

- 雙邊濾波（3次）：`O(3 × w × h × 81)`（9×9 核心）
- K-Means（20次迭代）：`O(20 × w × h × k)`
- 合計：`O(w × h × k)`

### 渲染合成

- 逐像素進行 bitwise_and 操作
- 複雜度：`O(w × h)`

整體時間複雜度：`O(w × h × k)` / 空間複雜度：`O(w × h)`

## 注意事項

1. 程式預設讀取同資料夾內的 `image.png`，請確認檔案存在或修改路徑。

2. 支援格式：任何 OpenCV 能讀取的格式（PNG、JPG、BMP、TIFF 等）。

3. K-Means 顏色數調整建議：
   - 預設 16 色：平衡卡通感與細節
   - 更多色彩（24～32）：更接近原始影像
   - 更少色彩（8～12）：更強卡通風格

4. 邊緣偵測參數調整建議：
   - 預設 block size=9：適合大多數影像
   - 更小（5～7）：偵測更細緻邊緣，可能增加雜訊
   - 更大（11～15）：粗糙邊緣，抗雜訊能力強

5. 若影像檔不存在或格式損壞，程式會拋出異常。

6. 首次執行可能較慢，因為 OpenCV 需要初始化及 K-Means 聚類需要迭代計算。
